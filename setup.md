# 반려동물 보호소 통합 탐색 및 실시간 알림 서비스 기획서

## 프로젝트 개요

### 프로젝트명
- findMyFriend

### 프로젝트 소개
전국 동물 보호소 및 동물 보호 단체 사이트의 보호 동물 공고를 통합 수집하여,
사용자가 찾고 있는 반려동물과 유사한 특징을 가진 동물이 등록될 경우 실시간 알림을 제공하는 서비스.

사용자는 매번 여러 사이트를 직접 방문하지 않아도 되며,
설정한 조건과 유사한 보호 동물이 새롭게 등록되면 자동으로 알림을 받을 수 있다.

---

# 프로젝트 목적

## 문제점

현재 잃어버린 반려동물을 찾기 위해서는:

- 여러 보호소 사이트를 직접 방문해야 함
- 사이트별 검색 UI가 다름
- 실시간 확인이 어려움
- 공고가 빠르게 누락될 수 있음
- 특징 검색이 제한적임

## 해결 목표

- 보호소 데이터를 통합 검색
- 사용자 맞춤 자동 탐색
- 실시간 신규 공고 감지
- 유사 특징 기반 알림 제공
- 보호 동물 발견 확률 향상

# 1. 보호 동물 통합 검색

## 검색 조건

### 발견 기간
- 기본값: 최근 3개월

- 조류
- 설치류
- 파충류
- 기타

### 성별
- 수컷
- 암컷
- 미상

### 중성화 여부
- O
- X

### 특징 검색

- Enter 키 검색 지원

# 개발 방향

본 프로젝트는 초기 MVP 단계에서
최소한의 기술 스택만 사용하여 빠르게 구현하는 것을 목표로 한다.

핵심 기능인:

- 보호소 데이터 수집
- 검색
- 유사도 분석
- 실시간 알림

구현에 집중하며,
복잡한 서버 구조나 불필요한 인프라는 최대한 제외한다.

---

# 기술 스택

# Frontend + Backend

## Next.js

Next.js(App Router)를 사용하여:

- 프론트엔드
- API 서버
- 서버 로직
- 크롤링 실행
- 알림 처리

를 하나의 프로젝트에서 통합 관리한다.

---

# Database

## SQLite

초기 MVP 데이터베이스는 SQLite를 사용한다.

### 선택 이유
- 별도 DB 서버 불필요
- 설정이 간단함
- 로컬 개발이 쉬움
- 빠른 MVP 개발 가능
- 추후 DB 이전 가능

---

# ORM

## Prisma

SQLite와 함께 Prisma ORM 사용.

### 사용 이유
- 타입 자동 생성
- 간단한 DB 관리
- 마이그레이션 관리 가능
- 추후 PostgreSQL 이전 용이

---

# 크롤링

## Playwright

보호소 사이트 크롤링을 위해 사용.

### 사용 이유
포인핸드와 같은 사이트는:
- JavaScript 렌더링
- 동적 데이터 로딩
- Lazy Loading

구조를 사용할 가능성이 높아
일반 fetch보다 안정적인 Playwright 사용.

---

# 배포

## Vercel

### 사용 이유
- Next.js 최적화
- 간단한 배포
- 빠른 CI/CD
- Cron 작업 사용 가능

---

# 제외한 기술 스택

초기 MVP에서는 아래 기술은 사용하지 않음.

## Redis
- 현재 규모에서는 불필요

## BullMQ
- Queue 시스템 필요성 낮음

## NestJS
- 현재 프로젝트 규모 대비 과한 구조

## Zustand
- React 기본 상태 관리로 충분

## React Query
- Next.js fetch 기반으로 충분
